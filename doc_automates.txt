------------------------------------------------------
 TABELA
------------------------------------------------------
 
t -> title (título do sitema)
 Ex: t:Controle de clientes
 
i -> insert (define se pode cadastrar na tabela)
 Ex: i:1
 Ex: i:0
 
m -> manage (define se pode gerenciar a tabela)
 Ex: m:1
 Ex: m:0
 
u -> update (define se pode atualizar dados da tabela)
 Ex: u:1
 Ex: u:0
 
d -> delete (define se pode deletar dados da tabela)
 Ex: d:1
 Ex: d:0
 
o -> order (define a ordem padrão da tabela)
 Ex: o:nm_campo,a ("a" para crescente)
 Ex: o:nm_campo,d ("d" para decrescente)
 
r -> relation (define a tabela que irá cadastrar dados com o código da tabela atual)
 Ex: r:tabela,titulo (ATENÇÃO, ESSE COMENTÁRIO É COLOCADO NO PRIMEIRO CAMPO DA TABELA ATUAL)
 
 
 
------------------------------------------------------
 CAMPOS
------------------------------------------------------
 
t -> title (título do campo)
 Ex: t:Nome
 
s -> size (tamanho do campo, caso não queira usar padrão)
 Ex: s:30
 
ml-> maxlength (tamanho máximo de caracteres no campo)
 Ex: ml:50
 
v -> validation (tipo de validação em tempo real do campo)
 Ex: v:number (somente números)
 Ex: v:email (somente emails válidos)
 Ex: v:url (somente urls válidas)
 
m -> mask (define uma máscara para o campo - "a" para letras, "9" para numeros e "*" para todos)
 Ex: m:99/99/9999
 Ex: m:99999-999
 Ex: m:(99) 9999-9999
 Ex: m:aaa-9999
 
l -> lista (define se o campo vai aparecer na lista de gerenciamento)
 Ex: l:1
 Ex: l:0
 
a -> align (define se o campo será centralizado na lista de gerenciamento)
 Ex: a:center
 
r -> relation (define a tabela que irá se relacionar)
 Ex: r:tabela,nm_campo,nm_campo2 ("nm_campo" que irá aparecer, e "nm_campo2" caso queira mostrar no gerenciar)
 
rc-> relation_codigo (define os campos que vão aparecer na janela externa relacionada com o código. Esta opção é usada junto com a r->relation)
 Ex: rc:nm_campo1,nm_campo2,nm_campo3
 
f -> file (define se o campo é pra enviar arquivos)
 Ex: f:jpg,gif
 
p -> padrão (define se o campo terá um valor padrão na hora de cadastrar)
 Ex: NOW()
 Ex: QUALQUERCOISA
 
o -> oculto (oculta o campo para não aparecer em telas de cadastro e alterações)
 Ex: o:1
 Ex: o:0
 
c -> campo (define o tipo de campo no html)
 Ex: c:radio
 Ex: c:checkbox,tabela,nm_campo ("nm_campo" que irá aparecer)
 Ex: c:password
 
ro-> readonly (define o campo como somente leitura)
 Ex: ro:1
 Ex: ro:0
 
ex-> exemplo (define se o campo vai ter algum tipo de exemplo de dados)
 Ex: ex:qualquercoisa
